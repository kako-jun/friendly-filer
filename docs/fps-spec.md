## 背景・コンセプト

友人 kako-jun が一貫して言っていた元ネタは映画『TRON』。「ファイルが人間になって現れる」世界観。これまでの「3D ファイラ」路線に **FPS レイヤ** を乗せ、ファイル操作のついでに遊べるものにする。

- **フィールド = カレントフォルダ**
- **敵（人型ワイヤーフレーム）= ファイル**
- **ポータル = サブフォルダ**
- **ディスクを投げて "識別"** → 操作メニュー（open / rename / move / copy / delete / info）
- **実用ファイラとして完結する**。FPS OFF モードも用意

## ビジュアル仕様（TRON 単一テーマ）

### パレット（3 色のみ）
- 背景: 黒〜濃い灰
- **青**（bright cyan、例 `#4FC3F7`）: グリッド線、壁縁、味方 UI、識別済みファイル
- **赤**（bright red、例 `#FF3D3D`）: 敵、警告、ロックオン、HP ゲージ
- 灰（`#3A3A3A` 〜 `#888888`）: ジオメトリの面

ターミナル発光は太字 + 高輝度 ANSI で代用。bloom / リガチャは不使用（既存方針）。

### オブジェクト → 表現

| 要素 | 表現 |
|---|---|
| 床 | 黒地に青グリッド線 |
| 壁・ポータル縁 | 青の輪郭線、面は灰 |
| ファイル（敵） | 人型ワイヤーフレーム、赤 |
| ポータル | 青リムの扉、内側は暗 |
| 石碑（現フォルダ自身の操作） | 青リムの立方体 |
| 親ゲート（..） | 背後の青帰還ライン（撃てない） |
| ロックオン | 赤の角括弧 `[ ]` |
| Swarm（群れ敵） | 赤の群体メッシュ |
| HUD | 全部青線画 |

## ゲーム性

### 武器: アイデンティティ・ディスク
- 投げる → **バウンド** → 複数敵をマルチヒット → **戻ってくる**
- 戻るまで次は投げられない（連射抑制）
- ディスクで削った HP = ロック強度、戻り時点で選択確定
- Enter / 右クリック = 操作発動

### プレイヤー移動
- WASD（or hjkl）移動
- **スペースでジャンプ**（単段、重力あり）
- 照準はキーボード + マウス両対応（config 切替、デフォルトはキーボード）

### 敵 AI
- ファイルタイプで挙動差分
  - `.rs` = 機敏、頻繁にジャンプ
  - `.log` = 重い、たまに跳ねる
  - `.png` = 浮遊気味
- 攻撃: 近接ランジ（体当たり）+ 一部敵は赤ディスク投擲
- 空中ヒット = スコアボーナス
- Swarm は波状ジャンプ

### HP（敵）
- `log(size)` ベース正規化、最大 5 発
- サイズの「硬さ」は装甲テクスチャ / 色で表現（実プレイ時間には反映しない）

### HP（プレイヤー）
- HUD 左上、青ゲージ
- HP 0 = **crash**: "SYSTEM ERROR" グリッチ 1 秒 → 現フォルダ入口に respawn、HP 全回復、ディスク即時復帰
- **ファイル状態には一切影響しない**。Game Over 無し

## LOD（ファイル数対策）

| ファイル数 | 表現 |
|---|---|
| 〜20 | 全敵個別描画 |
| 21〜100 | 個別 + 遠景薄描画、spawn は近接優先 |
| 100〜 | **Swarm 自動集約**（拡張子別・サイズ帯別）、1 群れ = 1 大型敵 |

Swarm は倒すと **bulk メニュー**（まとめて操作 / 分解して個別選択）。

## ポータル（サブフォルダ）

- **入る** = cd（降りる）
- **撃つ** = フォルダ自体の操作メニュー（copy / move / delete / rename / archive）
- HP = フォルダ total size（再帰、キャッシュ）
- 危険フォルダ（`/` や `$HOME` 直下）は **封印扉**、opt-in 確認経由でしか撃てない
- シンボリックリンクはループ検出で封印、扉点滅表示

## 操作メニューと演出

倒した瞬間 = **識別完了（IDENTIFIED）**: 赤 → 青フラッシュ、名前大表示、敵は直立のまま残る。**derezz は出さない**（破壊じゃない）。

メニュー選択後の演出:

| 操作 | 演出 |
|---|---|
| open | 扉状に開く → OS 既定アプリ起動 |
| rename | 名前プレート差し替え（旧名フェード、新名青点灯） |
| move | 青軌跡を引いてポータルへ吸い込まれる |
| copy | 青く分裂してもう 1 体生成 |
| delete | **derezz**（脱色 → 青粒子化）。この瞬間だけ TRON 消滅演出 |
| cancel | 赤に戻って再起動 |

delete は **OS ゴミ箱 / `.trash` に強制 move**。永久削除ではない。**`u` キーで Undo**（直前の操作巻き戻し）。

## サーチ / 入力モード

- `/` キー: 時空停止、fuzzy サーチ入力、ヒットへワープ or 矢印マーク
- rename / new file / new folder: 時空停止モード、安全にテキスト入力
- 石碑を撃つ = 現フォルダの操作メニュー（new file / new folder もここ）

## FPS OFF モード

- F1 で FPS 要素無効化: 敵は湧くが静止、照準のみで選択可能
- 純粋 3D ファイラとして使える気分の日用

## shell 統合

- `--cd-on-exit` で一時ファイル経由 cwd 出力、zsh / bash wrapper を README に提供
- 既存の ranger / yazi と同スタイル

## プレビュー

- ロックオン中 `Tab` で簡易プレビューサイドパネル（テキストは bat 風、画像は viu 風 or プレースホルダ）

## 外部変更監視

- `notify` crate で watch
- プレイ中にファイル増減 → ワープホール演出で敵 spawn / derezz
